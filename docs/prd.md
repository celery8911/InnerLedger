# InnerLedger PRD

# 📄 长了么  PRD

**协议名：InnerLedger**

**副标题**：AI 理解你，区块链记住你

**一句话定位**：用 AI 识别变化，用区块链记录过程

> 命名说明
> 
> - **「长了么？」** 是面向用户与评委的产品名称，一个反问，指向真实成长
> - **InnerLedger** 是底层协议名称，负责记录内在状态变化
> - **Growth SBT** 是 InnerLedger 生成的链上成长里程碑（不可转让）

---

## 产品愿景（Vision）

**「长了么？」** 是一个不会替你思考的AI+web3系统，它只帮你看见自己。

我们希望构建一个系统，让用户：

- 在冥想与觉察中与自己相处
- 被 AI 理解与映照，而非指导与评判
- 将真实发生过的内在变化，记录为不可转让、不可篡改的成长凭证

## 三、核心功能（长了么 App）

### 3.1 冥想体验

**模式**：5 分钟呼吸冥想（MVP）

**交互**：

- 极简界面，仅呼吸动画
- 无倒计时压力，可随时退出
- 结束后只问一句：
    
    **“刚才，有什么被你注意到了？”**
    

**特点**：

- ❌ 无打卡、无积分、无排行
- ✅ 可跳过觉察输入
- ✅ **每一次冥想都会被 InnerLedger 记录为链上过程**

---

### 3.2 觉察记录

**输入方式**：

- 冥想后自动进入
- 或任意时间添加反思

**上链流程**：

```
用户输入 → 本地加密 → 生成哈希 → AI 分析 → 上链
 ↓
提示：「✅ 已永久记录（查看链上证明）」

```

---

### 3.3 AI 分析（理解层）

**MVP 简化版**：

**用户参与 + AI 辅助**的混合模式

1. **用户自评情绪**（必选）
   - 冥想后选择情绪：😊 平静 / 😐 中性 / 😔 不安

2. **AI 复述理解**（可选）
   - 输入："今天注意到呼吸变慢了"
   - AI 回应："你觉察到了呼吸的变化"
   - 目的：让用户感到被理解，而非被分析

3. **链上元数据**（自动生成）

| 字段 | 说明 | 生成方式 |
| --- | --- | --- |
| contentHash | 内容哈希 | keccak256(原文) |
| emotion | 情绪标签 | 用户选择 |
| timestamp | 时间戳 | 自动 |
| userId | 钱包地址 | 自动 |

（⚠️ 原始文本永不上链，存储在本地）

**未来规划**：
- 情绪趋势分析（基于历史数据）
- 觉察深度评分（基于文本长度和关键词）
- 变化检测（对比前后记录）

---

### 3.4 成长时间轴（长了么 · Timeline）

展示的是**过程日志**，而不是成就：

- 冥想 / 反思记录
- 情绪趋势
- 觉察热力日历
- 每一条都可点击「链上证明」

### 3.5 Growth SBT（成长里程碑）

> Growth SBT = 被见证过的成长阶段快照
>

**MVP 里程碑**（简化版）：

| 里程碑 | 条件 | SBT 名称 | MVP 状态 |
| --- | --- | --- | --- |
| 1 | 首次记录 | "Journey Begins" | ✅ 必做 |
| 2 | 7 次记录 | "First Week" | ⚠️ 可选 |
| 3 | 30 次记录 | "One Month" | ⚠️ 可选 |

**未来扩展**：
- 情绪变化里程碑（基于趋势）
- 连续打卡里程碑（坚持 30/60/100 天）
- 深度觉察里程碑（长文本记录）

**SBT 特性**：
- 不可转让（SoulBound）
- 不可交易
- 永久记录在链上
- 每个 SBT 包含：
  - 用户地址
  - 达成时间
  - 里程碑类型
  - 唯一 Token ID

**核心理念**：

**「有一件事，和以前不一样了。」**

## 四、智能合约（InnerLedger）

### 4.1 InnerLedger 合约（核心记录）

**主要功能**：

```solidity
contract InnerLedger {
    struct Record {
        bytes32 contentHash;      // 内容哈希
        uint256 timestamp;        // 记录时间
        string emotion;           // 情绪标签
        address user;             // 用户地址
    }

    mapping(address => Record[]) public journeys;

    // 创建记录
    function createRecord(bytes32 _contentHash, string memory _emotion) external;

    // 获取用户所有记录
    function getJourney(address _user) external view returns (Record[] memory);

    // 获取用户记录数量
    function getRecordCount(address _user) external view returns (uint256);
}
```

**事件**：
```solidity
event RecordCreated(address indexed user, bytes32 contentHash, uint256 timestamp);
```

---

### 4.2 GrowthSBT 合约（里程碑证明）

**基于 ERC-721 的不可转让版本**：

```solidity
contract GrowthSBT is ERC721 {
    struct Milestone {
        string name;          // 里程碑名称
        uint256 achievedAt;   // 达成时间
        address user;         // 用户地址
    }

    mapping(uint256 => Milestone) public milestones;
    uint256 public nextTokenId;

    // Mint SBT（只能由 InnerLedger 调用）
    function mintGrowthSBT(address _user, string memory _milestoneName) external;

    // 重写 transfer 函数，禁止转让
    function _transfer() internal pure override {
        revert("SoulBound: Transfer not allowed");
    }

    // 获取用户的所有 SBT
    function getUserSBTs(address _user) external view returns (uint256[] memory);
}
```

**触发逻辑**（在 InnerLedger 中）：
```solidity
function createRecord(...) external {
    // ... 创建记录

    // 检查里程碑
    uint256 count = journeys[msg.sender].length;
    if (count == 1) {
        growthSBT.mintGrowthSBT(msg.sender, "Journey Begins");
    } else if (count == 7) {
        growthSBT.mintGrowthSBT(msg.sender, "First Week");
    }
}
```

---

### 4.3 部署顺序

1. 部署 GrowthSBT 合约
2. 部署 InnerLedger 合约（传入 GrowthSBT 地址）
3. 设置 GrowthSBT 的 minter 权限给 InnerLedger

### 5.3 AI 理解层（核心）

**这是产品的灵魂，必须实现**

| 技术 | 用途 | 实现方式 |
| --- | --- | --- |
| **OpenAI GPT-4o-mini** | 文本理解与复述 | **P0 必做** |
| Next.js API Routes | AI 调用封装 | P0 |
| 流式响应 | 打字机效果 | P1 |

**AI 的三个职责**（符合产品理念）：

1. **复述**（P0）
   - 用户输入："今天注意到呼吸变慢了"
   - AI 复述："你觉察到了呼吸的变化"
   - Prompt: "用温和的语气复述用户的觉察，不做评判，不给建议"

2. **提问**（P1）
   - 根据用户输入，提出一个轻微的正念式问题
   - 例如："这个变化让你感受到了什么？"
   - Prompt: "基于用户的觉察，提出一个开放式的探索问题"

3. **留白**（P0）
   - 当用户的觉察已经很完整时，AI 只需简短确认
   - 例如："嗯，我听到了"
   - Prompt: "判断用户是否需要回应，必要时保持简洁"

**技术实现**：
```javascript
// app/api/ai/understand.ts
export async function POST(req: Request) {
  const { userInput } = await req.json();

  const response = await openai.chat.completions.create({
    model: "gpt-4o-mini",
    messages: [
      {
        role: "system",
        content: "你是一个温和的陪伴者，只复述和理解，不评判不指导。"
      },
      {
        role: "user",
        content: userInput
      }
    ]
  });

  return Response.json({ aiResponse: response.choices[0].message.content });
}
```

**成本控制**：
- GPT-4o-mini：$0.15/1M tokens（非常便宜）
- 单次调用约 100 tokens = $0.000015
- 1000 次用户输入 ≈ $0.015

### 6.3 Gas 费优化方案

**问题**：每次上链虽然便宜（$0.007），但累积起来仍有成本

**解决方案**（选其一）：

| 方案 | 实现 | 适用场景 |
| --- | --- | --- |
| **Gasless 交易** | 使用 ERC-2771（Meta Transaction） | P0 推荐 |
| **批量上链** | 每周提交一次 Merkle Root | P1 备选 |
| **重要时刻上链** | 用户选择哪些记录上链 | P2 备选 |

**MVP 推荐**：
- 使用 Gasless 方案，Demo 中强调"用户无需支付 gas"
- 或者预先给测试账户充值 $1，足够 100+ 次记录

### ① Opening（30 秒｜讲故事）

今天，大多数 Web3 项目都在讨论资产、金融、收益，涨了么？

但我想先问一个问题：

> 如果我们拥有资产，却失去了内在的稳定，那些资产还有意义吗？
> 

> 长了么 在意的不是你的外在资产的涨跌，而是关心你的内在是否安宁、幸福，你的心灵有无变化、成长？
> 

我们每天在 Web2 平台上：

- 记录情绪
- 学习冥想
- 写成长日记

但这些**最私密、最重要的心灵数据**，

**并不真正属于我们自己。**

### ② Problem（问题｜30 秒）

当前的心灵成长产品存在三个核心问题：

1. **数据不属于用户**
    - 情绪、反思、成长轨迹被平台掌控
2. **成长没有“长期身份”**
    - 连续冥想 100 天，换手机、换 App，一切归零
3. **激励是外在的**
    - 打卡为了奖励，而不是内在改变

👉 心灵成长，本质上是**人格资产**，

但目前它是**无主的、不可迁移的、不可验证的**。

---

### ③ Solution（解决方案｜45 秒）

我们的产品叫 **「长了么」**，

它背后的协议叫 **InnerLedger**。

> AI 是这个系统的“陪伴与理解层”，
区块链是“过程记忆层”。
> 

用户通过钱包登录，每天进行：

- 冥想
- 反思
- 觉察记录

在「长了么」里，

**AI 不做心理诊断，也不替你思考。**

它只通过三种方式陪伴你：

- **复述**：帮你听见自己
- **提问**：轻轻指向一个你可能忽略的角度
- **留白**：在没有必要的时候，什么也不说

**与此同时，每一次冥想和觉察，都会被 InnerLedger 记录为一次链上过程。**

> 我们不上链原始内容，
只上链这一刻的元数据与时间戳，
记录的是“发生过一次觉察”，而不是你说了什么。
> 

AI 会在后台判断一件事：

> 现在的你，
和过去的你有没有出现新的变化。
> 

当这些变化在时间中持续出现，

InnerLedger 会在关键阶段生成一个 **SoulBound Token**：

- 不可交易
- 不可转让
- 只代表一件事：
    
    **这一整段成长历程，被完整见证过。**
    

---

### ④ Why Web3 + AI（为什么一定要 Web3｜45 秒）

这个项目**不能用 Web2 实现**。

- **SoulBound Token**
    - 成长不能被买卖
- **钱包即身份**
    - 不依赖手机号或平台账号
- **用户拥有数据主权**
    - 内容可加密
    - 只公开“成长证明”，不公开隐私

一句话总结：

> Web2 记录你的行为，Web3 记录你成为了谁。
>

### 9.3 评委可能的质疑

**Q1: "为什么不直接在 Web2 做？"**

**A**:
- Web2 无法提供不可篡改的时间戳
- Web2 数据不属于用户（平台倒闭即清零）
- SBT 的社会价值需要区块链的可验证性

**Q2: "用户凭什么相信 AI 理解是准确的?"**

**A**:
- AI 的职责不是"诊断"而是"陪伴"
- 用户可以参与情绪标注
- 重点是记录过程，而非评判对错

**Q3: "如何防止用户刷数据获得 SBT?"**

**A**:
- MVP 阶段暂不解决（诚实使用）
- 未来可以加入：
  - 文本质量检测
  - 时间间隔限制
  - 社区举报机制

**Q4: "Gas 费累积起来还是很贵？"**

**A**:
- 展示成本对比表（$0.70 vs $200）
- 说明 Gasless 方案
- 强调 Monad 的不可替代性

**Q5: "如何获取第一批用户？"**

**A**:
- Web3 社区冥想爱好者
- DAO 成员寻求人格证明
- 关注心理健康的 KOL
